# AGENTS 指南（中文）

本文件用于统一 AI 代理在本仓库内的工作方, 式目标是让不同编辑器或不同 AI 工具都能遵守同一套规范

## 语言与沟通

- 默认使用中文输出
- 回答应简洁、可执行，避免空泛描述
- 在不确定需求时先澄清，再实施
- 大需求先给方案

## 修改流程

- 修改前先阅读相关上下文，避免破坏现有行为
- 变更尽量小步提交，避免无关重构混入
- 完成后如果需要就执行可用的 lint / test / build 校验

## 安全与边界

- 不提交密钥、令牌、凭证或其他敏感信息
- 不执行破坏性操作（如强制覆盖、无确认删除）除非用户明确要求
- 不回退或覆盖与当前任务无关的现有修改

## 代码约定

- 所有代码必须使用 TypeScript 编写
- 使用 pnpm 管理依赖
- early return 风格
- 优先编写职责单一、可读性强的小函数
- 优先写将大 react 组件拆分写成 小组件, 放在同一文件夹内
- 所有文件名与文件夹名必须使用 kebab-case 风格
- 不要用 tailwind.css, 参考其他用 linaria css 写法
- 类型可自动推导时不要显式标注
- 简化箭头函数简化写法, 适当压缩代码行数
- 写代码时主动思考是否有更好的写法, 优先复用项目已有函数、类型与抽象, 避免重复实现
- 优先使用 @gitborlando/utils 中的工具函数
- 优先使用 @gitborlando/geo 中的类型以及几何计算
