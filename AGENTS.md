# AGENTS 指南（中文）

本文件用于统一 AI 代理在本仓库内的工作方, 式目标是让不同编辑器或不同 AI 工具都能遵守同一套规范

## 语言与沟通

- 默认使用中文输出
- 回答应简洁、可执行，避免空泛描述
- 在不确定需求时先澄清，再实施
- 大需求先给方案

## 修改流程

- 修改前先阅读相关上下文，避免破坏现有行为
- 变更尽量小步提交，避免无关重构混入
- 完成后如果需要就执行可用的 lint / test / build 校验
- 执行前需在 ai-plans 文件夹按提示摘要添加记录, 格式为 <摘要(简)>-<日期(简)>/<步骤(中文)>.md

## 安全与边界

- 不提交密钥、令牌、凭证或其他敏感信息
- 不执行破坏性操作（如强制覆盖、无确认删除）除非用户明确要求
- 不回退或覆盖与当前任务无关的现有修改

## 代码约定

- 所有代码必须使用 TypeScript 编写
- 使用 pnpm 管理依赖
- 优先编写职责单一、可读性强的小函数
- 优先写将大 react 组件拆分写成 小组件, 放在同一文件夹内
- 所有文件名与文件夹名必须使用 `kebab-case`
- 不要用 tailwind.css
- 类型可自动推导时不要显式标注
- 箭头函数简化写法, 如: `(a: number) => { return a + 1 }` 可以简写为 `a => a + 1`
- 优先使用 @gitborlando/utils 中的工具函数
- 优先使用 @gitborlando/signal 中的信号管理
- 优先使用 @gitborlando/geo 中的类型以及几何计算
