# AGENTS 指南（中文）

本文件用于统一 AI 代理在本仓库内的工作方式。目标是让不同编辑器或不同 AI 工具都能遵守同一套规范。

## 1. 语言与沟通

- 默认使用中文输出。
- 回答应简洁、可执行，避免空泛描述。
- 在不确定需求时先澄清，再实施。

## 2. 命名规范（强制）

- 所有文件名与文件夹名必须使用 `kebab-case`。
- 仅允许小写字母、数字和 `-`。
- 禁止使用 `camelCase`、`PascalCase`、`snake_case`。

示例：

- 正确：`user-profile.ts`、`auth-service.ts`、`user-management/`
- 错误：`userProfile.ts`、`AuthService.ts`、`user_management/`

## 3. 代码约定

- 优先编写职责单一、可读性强的小函数。
- React 组件保持聚焦，复用逻辑提取为 hooks 或工具函数。
- 对外暴露的函数参数和返回值使用明确类型。
- 禁止空 `catch`；错误需要显式处理或上抛。
- 避免魔法数字与含义不清的缩写命名。
- 所有代码必须使用 TypeScript 编写。
- 使用 pnpm 管理依赖。

## 4. 修改流程

- 修改前先阅读相关上下文，避免破坏现有行为。
- 变更尽量小步提交，避免无关重构混入。
- 完成后执行可用的 lint / test / build 校验。
- 若本地无法执行校验，需要在说明中明确未验证项与建议验证命令。

## 5. 安全与边界

- 不提交密钥、令牌、凭证或其他敏感信息。
- 不执行破坏性操作（如强制覆盖、无确认删除）除非用户明确要求。
- 不回退或覆盖与当前任务无关的现有修改。

## 6. 与 Cursor Rules 的关系

- 本文件是跨编辑器的通用规范入口。
- 若两者存在冲突，以用户明确指令为最高优先级，其次以更严格约束为准。

## 7. 偏好配置

- 使用 @gitborlando/utils 中的工具函数。
- 使用 @gitborlando/signal 中的信号管理。
- 使用 @gitborlando/geo 中的类型以及几何计算。
- 类型可自动推导时不要显式标注。
- 箭头函数简化写法, 如: `(a: number) => { return a + 1 }` 可以简写为 `a => a + 1`。
